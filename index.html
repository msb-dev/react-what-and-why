A brief history of the web

Invented by Tim Berners-Lee in 1989 at CERN

Designed for publishing and linking articles

Berners-Lee created:

  • a web server
  • a web browser
  • HTTP to communicate
  • HTML

Static pages

HTML files were stored on web servers

A server could be browsed like a file system

Content was static and read-only

Life was simple! 😀

Server-side rendering

Later, user generated data was made possible

Data could be entered by the user via "forms"

The server would run code to generate dynamic pages based on the user's data

Problem: full page refreshes

Javascript

With the dawn of JavaScript, the web became more interactive

XMLHttpRequest allowed JS to get and send data to the server

Now we could update a page without a full refresh

Browsers with JavaScript became widely used

Manipulating an element


            // Get a reference to the element you want to modify
            const textbox = document.getElementById('query-textbox');
            // or e.g. document.querySelector('textbox.someClass')

            // Mutate the `value` or `innerHTML`

Adding an element


            // Create the element you want to insert
            const listItem = document.createElement('li');
            listItem.value = 'Some text';

            // Get the container you want to insert new element into
            const list = document.getElementById('main-list');

            // Append the element to the container
            list.append(listItem);

Props

Props allow us to pass data into a component

  • Point out that React components are (almost) just functions
  • Functions are useful because the code they contain can be defined once and used multiple times
  • Just like functions can take arguments to make them more flexible and dynamic, so can components (but a component takes a single object argument which we add properties ["props"] to)

State

State is a way to store data in React

State gets passed down (via props) into child components; it can't be passed up the tree

  • useState returns an array. First element is the state itself; second element is a function you can call to set the state.
  • Always set state using the setter so React knows the state has changed and that it should call your componet functions again to see if anything needs rerendering
  • When state changes and React calls your component functions, these return what the DOM elements on your page should look like now
  • React compares this "desired state" with what's currently in the DOM and then only changes the DOM elements that need changing to bring the DOM into line.

Lifting up state

State is always "owned" by a single component

If you need state higher up, move the useState to a parent or grandparent component

Only lift it as high as it needs to go
