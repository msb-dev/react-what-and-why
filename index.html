A brief history

Computers were huge and there were not many of them

Scientists needed a way to access informaiton on them

Created in 1989 by Tim Berners-Lee whilst working at the CERN physics laboratory

File stores

HTML documents being served by servers

Viewed as a file system through early browsers

Server side rendeing

User generated data was made possible

Data could be enterd by the user. The server would return a site that would use this entered data

Problem: full page refreshes

Javascript

Websites became interactive. JS can edit the DOM directly within browsers

XMLHttpRequests could make requests to get data from servers

User generated data was made possible without full page refreshes

Browser Javascript engines became widely used and more powerful

Components

Components can split up the logic within the page into smaller, named, manageable chunks

Props

Props allow us to pass data into a component

State

State is a way to house variables in React

State gets passed down into child components, it cannot be passed up the tree

Lifting up state

State is always "owned" by a single complement

If you need state higher up, move the useState to a parent or grandparent component

Only lift it as high as it needs to go
